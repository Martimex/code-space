
Pre-trained models are types of models that have already been trained by someone else, so that there is no necessity to gather own data or spending time and resources to train model yourself.
Such models can be easily loaded and used directly for the task they were trained for within your own production system.

## TFJS external models 

Pre-trained TensorflowJS models are being continuously developed by independent researchers, professional developers, etc. Models coming outside of TFJS team might require more machine learning knowledge in order to use them. It is because the models come in their raw form, with no easy-to-use helper functions (wrapper classes).   

## TFJS custom models

There are plenty of custom TensorflowJS models that are curated by TensorflowJS team. These models are wrapped in easy-to-use Javascript classes, that can be used with just few lines of code.  
Custom models are great for people new to machine learning.

Below is the list of use-case categories with appropiate custom models assigned to each of them.

### Vision models

* Image classification
* Object detection  =>  **COCO SSD**

### Human body

* Body segmentation  =>  **MediaPipe Pose**,  **Bodypix Pose**
* Pose estimation  =>  **MediaPipe**,  **MoveNet**,  **BlazePose**
* Face Landmark Detection
* Hand Pose Estimation

### Text

* Text toxicity
* Sentence encoding
* BERT Q&A
* Conversation Intent Detection

### Sound

* Sound recognition

### Other

* KNN classifier


## Example usage (custom TFJS)

In terms of custom TFJS models (ones curated by TFJS team), their usage comes down to few essential steps. Let's examine the process following the BERT Q&A model insertion:

```
[1]
<script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs"></script>
<script src="https://cdn.jsdelivr.net/npm/@tensorflow-models/qna"></script>

<script>
   [2]
   const searchText = "We believe cats are the real stars of YouTube";
   const question = "What is important to YouTube?";

   [3]
   qna.load().then(model => {
      [4]
      model.findAnswers(question, searchText).then(answers => {
         console.log('Answers: ', answers);  // Will output: "Answers: Cats"
      });
   });
```

[1] -> First of all - imports. The very first line installs TensorflowJS library, while the following installs the specifc model we plan to use (in this scenarion, it is BERT Q&A).

[2] -> Next up, we need to define variables (input values) that are required to be used inside a model. In this example `searchText` and `question` variables are defined.

[3] -> Using the load() function, a `qna` model is being loaded asynchronously.

[4] -> Once the `qna` model has finished loading, the implemented model methods can be used, such as `findAnswers()`, which takes input parameters (`question` and `searchText`) and produces the output (`answers`), which is the answer to the provided question found in the searchText.
